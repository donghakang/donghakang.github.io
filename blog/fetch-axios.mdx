# fetch 와 axios의 차이점

웹을 개선하면서 HTTP 통신은 뭐 당연히 필요한 항목중 하나고, 프론트엔드 개발자로서 HTTP 통신에 대해서 앞으로도 많이 접해볼 계기가 있는 것은 당연하다.

fetch를 주로 사용했는데, 뭐 어쩔때는 axios를 사용하고..

근데 둘의 차이점이 뭐야? 

## 둘의 차이점

기본적인 개념은 아주 비슷하게 흘러가지만 아래와 같은 내용들이 차이점이다

 

| axios | fetch |
| --- | --- |
| 3rd party 패키지이기에 npm install을 해야한다 | javascript에 탑재 되어있다 |
| axios는 XSRF 보호가 탑재되어있다 (XSRF 란, 쿠키만으로 인증하는 서비스에 특정 명령을 요청해서 공격하는 방식) | XSRF를 따로 보호하지 않는다
 |
| data 프로퍼티를 사용한다 | body 프로퍼티를 사용한다 |
| data는 object를 갖고있다 | fetch의 body는 stringified 되어야한다. |
| status 200 이며 statusText 가 ‘OK’ 일 때 통신 성공이다 | response.ok 로 통신을 확인한다 |
| 자동으로 JSON 형식으로 전환한다 | 먼저 JSON data 를 불러온 다음 그 다음에 res.json()으로 변환해야한다. |
| request cancelling 과 request timeout이 가능하다 | X |
| HTTP request를 인터셉션할 수 있다. | X |
| download progress를 확인할 수 있다. | X |
| 많은 브라우저에 사용 가능하다. | Chrome 42+, Firefox 39+, Edge 14+, Safari 10.1+ 에서만 사용할 수 있다.  |
