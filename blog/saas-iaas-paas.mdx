위 용어들은 클라우딩 컴퓨팅이 도입되면서 생겨난 용어들인데, 예전 같은 경우 IT 인프라의 필요한 구성요소를 모두 사용자가 (인프라 구축) 관리 해야했지만, 이제는 클라우드에서 내려받을 수 있는 형태가 되었다.

예전 같은 경우 (Packaged Software), 물리 적인 장치, 하드웨어 (CPU, RAM, Storage, Network device) 을 모두 직접 구매하고 OS 를 설치한뒤 서버 관리를 직접적으로 해야했었다. 하지만 IaaS, PaaS, SaaS 가 나오면서 이는 불필요하게 변경 되었다.

![스크린샷 2022-05-30 오후 4.09.48.png](/img/blog/saas.png)

## IaaS

특징

- 기업이 준비해놓은 가상환경에서 장비 구매 필요없이 호스팅을 선택할 수 있다. 대표적으로 AWS
- OS 어플리케이션을 직접 관리할 수 있다.
- 개발자 / 인프라 구축을 따로 분리 가능하다.
- 가상 서버 하위 레벨에 대해서 고려할 필요가 없는 사용자가 쓰기 적합하다.

예시

- AWS의 EC2 (물리적인 서버와 Network, Storage없이 원하는 OS를 다운받아 서버로 사용 가능하다)

## PaaS

특징

- OS, Server 하드웨어, Network 고려할 필요가 없음
- 어플리케이션 자체에만 집중할 수 있게 해줌.
- PaaS는 node.js, Java와 같은 런타임을 미리 깔아놓고, 소스코드를 넣어서 돌리는 구조. 소스코드만 적어서 빌드하고 클라우드에서 결과만 가져온다.

예시

- Heroku, Google App Engine, IBM Bluemix, OpenShift, SalesForce 등..

## SaaS

특징

- 설치할 필요도 없고 클라우드를 통해 제공
- 트래픽 기반으로 돈을 벌고 구독 방식으로 돈을 범
- 매우 쉽고 간편함

예시

- 웹 메일, 구글 클라우드, vercel, netlfiy 등…
