## Screen

```css
@media screen and (orientation: landscape) {
  body::after {
    content: "Landscape";
  }
}

@media screen and (orientation: portrait) {
  body::after {
    content: "Portrait";
  }
}
```

```css
// use this when it is image based site
@media (max-width: 400px) {
  // Styles for viewports narrower than 400 pixels.
}

// use this when it is text based site
@media (min-width: 25em) {
  // Styles for viewports wider than 25em.
}
```

## Layout

레이아웃은 크게 두가지로 구분해서 디자인 하면 좋다. Macro 와 Micro Layout.

Macro 같은 경우 큰 레이아웃 즉 컴포넌트의 위치를 구성할 때 사용하면 좋고, Micro는 컴포넌트 안에서의 위치 이미지 위치 텍스트 위치 등 세분화된 구성요소를 구성할 때 사용한다.

### Macro

컴포넌트가 많으면 많을 수록 큰 틀을 Grid를 이용해서 만들어주자

```css
@media (min-width: 45em) {
  main {
    display: grid;
    grid-template-columns: 2fr 1fr;
  }
}
```

### Micro

컴포넌트 안에서는 grid 도 좋지만 flex box 도 잘 사용해보자

```css
.media {
  display: flex;
  align-items: center;
  gap: 1em;
}
.media-illustration {
  flex: 1;
  max-inline-size: 200px;
}
.media-content {
  flex: 3;
}
```

## Typography

화면이 작으면 작을수록, 크면 클수록 텍스트의 크기를 변경하는것은 당연하다.

```css
html {
  font-size: calc(0.75rem + 1.5vw);
}
```

⚠️ 직접적으로 vw 를 사용하면 resize 하기에 너무나도 어렵다.

```css
html {
  font-size: clamp(1rem, 0.75rem + 1.5vw, 2rem);
}
```

clamp를 사용해서 최소의 텍스트 사이즈와, 최대의 텍스트 사이즈를 미리 정해버리자.

max-inline-size 같은 경우도 텍스트 사이즈에 따라서 많이 변경되므로, px단위로 정하지 말자.

```css
article {
  max-inline-size: 66ch;
}
```

line-height은 inline-size와 비례하게끔, px단위로 정하지 말자.

```css
line-height: 1.5;
```

## 추가적인 기능

항상 overflow-x: hidden으로 정해놓고 옆으로 스크롤 하는 기능을 만들었었는데 사실 carousel이라는 기능이 있다.

```css
@media (max-width: 50em) {
  .cards {
    display: flex;
    flex-direction: row;
    overflow-inline: auto;
    scroll-snap-type: inline mandatory;
    scroll-behavior: smooth;
  }
  .cards .card {
    flex-shrink: 0;
    flex-basis: 15em;
    scroll-snap-align: start;
  }
}
```
